PGE-SC • Hub de Ferramentas

Plataforma PGE

Centraliza ferramentas jurídicas e administrativas (pautas de audiência, atos normativos, reservas e integrações). Autenticação via JWT/LDAP, backend Django + PostgreSQL, frontend Next.js com design Radix/shadcn.

Destaques

Hub unificado

Dashboard com busca, categorias, favoritos e notificações para acessar ferramentas (Pautas, Atos, Reservas, Miner).

Pautas de audiência

Filtros avançados (órgão, relator, assunto, tipo de sessão), ranking por câmaras e painel consolidado.

Atos normativos

Consulta e gestão de atos com listagens, filtros e UI responsiva baseada em Radix.

Reserva de salas

Agenda de salas/auditórios com status em tempo real (integração Firebase) e verificação de conflitos.

Segurança e SSO

Autenticação via JWT com cookies httpOnly, suporte opcional a LDAP corporativo e controle de CORS/CSRF.

Integrações

Webservices SOAP parametrizados (Zeep/xmltodict), cadastro de serviços/tribunais, manipulação de envelopes, e-mail e logs estruturados.

Stack e Arquitetura

Backend

Django 3.12+, DRF, SimpleJWT (cookies httpOnly), CORS, WhiteNoise, logging estruturado.

Django DRF SimpleJWT CORS WhiteNoise

Frontend

Next.js + React 18, TypeScript, Radix UI/shadcn, lucide-react, react-hook-form, recharts, embla.

Next.js React 18 TypeScript Radix UI shadcn

Dados & Integrações

PostgreSQL 16, Zeep/XML (SOAP) com WSDLs e parâmetros dinâmicos, xmltodict, integração Firebase para reservas em tempo real.

PostgreSQL Zeep/SOAP xmltodict Firebase (reservas)

Infra/DevOps

Docker Compose (dev/prod), imagens separadas de frontend/backend, env files versionados por ambiente.

Docker Compose .env por ambiente
Galeria
Como rodar (dev)
Docker Compose (dev)
`docker-compose -f docker-compose-dev.yml up` (sobe Postgres, backend e frontend).
Backend
DJANGO_SETTINGS_MODULE=config.settings.development; migrações automáticas no entrypoint; API em 8001.
Frontend
NEXT_PUBLIC_API_BASE_URL=http://localhost:8001; app Next em 3001.
Produção
`docker-compose -f docker-compose-prod.yml up -d` com envs em `infra/.env/.env.prod`.
